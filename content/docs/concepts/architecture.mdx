## Overview

Zylora is a serverless GPU runtime designed for enterprise workloads. Functions are deployed as containers and executed on demand across a multi-provider compute network.

```
┌─────────────────────────────────────┐
│           Your Code                 │
│  @zylora.function / SDK / CLI       │
└──────────────┬──────────────────────┘
               │ HTTPS
┌──────────────▼──────────────────────┐
│         Zylora API Gateway          │
│   Auth · Rate Limit · Request ID    │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│         Core Engine (Rust)          │
│  Router · Scheduler · Metering      │
└────┬──────────────────────┬─────────┘
     │                      │
┌────▼────────────────────────────────┐
│    Multi-Provider GPU Compute       │
│  Capacity · Failover · Placement    │
└─────────────────────────────────────┘
```

## Request Lifecycle

1. **API Gateway** receives the invocation request, validates auth, applies policy checks, and attaches tracing metadata.
2. **Routing layer** selects an execution target using availability, policy, and latency objectives.
3. **Execution control plane** allocates runtime capacity and routes to warm or newly started workers.
4. **Runtime** executes the function, supports streaming responses, and emits usage telemetry.
5. **Metering** records billable usage for cost visibility and invoicing.
6. **Response** returns output with timing and request metadata.

## Performance Strategy

Zylora uses multiple warm-path techniques to reduce startup latency and improve p95 response times for repeated workloads. Runtime state is version-aware and safely refreshed on redeploy.

## Provider Abstraction

Compute providers are integrated behind a common abstraction layer. This allows controlled routing, policy enforcement, and transparent failover without changing user-facing APIs.

## Reliability & Data Guarantees

- All entity IDs are UUID v7 (time-sortable, no hotspots)
- All timestamps are UTC with timezone info (`TIMESTAMPTZ`)
- Usage events are append-only with partition pruning by month
- Billing is in integer cents — no floating point
- Multi-provider failover is automatic when capacity or health conditions change
