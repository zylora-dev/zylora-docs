## Overview

A Zylora function is a Python function decorated with `@zylora.function`. When deployed, it runs inside a Docker container with GPU access on Zylora's infrastructure.

## Step 1: Define the Function

```python
# whisper_transcribe.py
import zylora

@zylora.function(
    gpu="A10G",
    memory_gb=24,
    timeout_secs=120,
)
def transcribe(audio_url: str) -> dict:
    import whisper
    import urllib.request

    model = whisper.load_model("base")
    urllib.request.urlretrieve(audio_url, "/tmp/audio.mp3")
    result = model.transcribe("/tmp/audio.mp3")
    return {"text": result["text"], "language": result["language"]}
```

### Decorator Options

| Option | Type | Default | Description |
|---|---|---|---|
| `gpu` | `str` | `"A10G"` | GPU type — see [GPU Types](/docs/concepts/gpu-types) |
| `memory_gb` | `int` | `24` | GPU VRAM in GB |
| `timeout_secs` | `int` | `300` | Max invocation wall time |
| `cpu` | `int` | `4` | vCPU count |
| `ram_gb` | `int` | `16` | System RAM in GB |
| `retries` | `int` | `0` | Auto-retry on failure |
| `secrets` | `list[str]` | `[]` | Secrets to inject as env vars |

## Step 2: Add Dependencies

Create `requirements.txt` next to your function:

```
openai-whisper==20231117
torch==2.2.0
```

Or use a `pyproject.toml` with optional dependencies — Zylora auto-detects both.

## Step 3: Deploy

```bash
zylora deploy whisper_transcribe.py
```

Watch the build log stream in your terminal:

```
[1/5] Resolving dependencies...
[2/5] Building Docker image...
[3/5] Installing packages (torch, openai-whisper)...
[4/5] Pushing to registry...
[5/5] Deploying function...
✓ my-org/transcribe deployed in 68s
  Endpoint: https://api.zylora.dev/v1/functions/my-org/transcribe/invoke
```

## Step 4: Invoke

```bash
zylora invoke my-org/transcribe \
  --input '{"audio_url": "https://example.com/recording.mp3"}'
```

## Step 5: Monitor

View invocation logs in the [Dashboard](/docs/concepts/architecture) or via CLI:

```bash
zylora logs my-org/transcribe --tail
```
