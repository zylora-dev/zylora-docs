## Overview

By default, Zylora auto-builds a Docker image from your Python file. For complex dependencies (CUDA extensions, custom drivers, non-Python runtimes), you can provide your own `Dockerfile`.

## Using the Zylora Base Image

The Zylora runner base image provides:
- CUDA 12.4 + cuDNN 9
- Python 3.11
- The `zy_runner` process supervisor

```dockerfile
FROM ghcr.io/zylora-dev/runner:cuda12.4-py311

# Install system deps
RUN apt-get update && apt-get install -y ffmpeg libsndfile1

# Install Python deps
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy your function
COPY inference.py .
```

## Specifying the Dockerfile

```python
import zylora

@zylora.function(
    gpu="A100-80GB",
    dockerfile="./Dockerfile",  # relative to the Python file
)
def run(prompt: str) -> str:
    ...
```

Or via CLI:

```bash
zylora deploy inference.py --dockerfile ./Dockerfile
```

## Available Base Images

| Tag | CUDA | Python | Notes |
|---|---|---|---|
| `cuda12.4-py311` | 12.4 | 3.11 | Recommended |
| `cuda12.1-py310` | 12.1 | 3.10 | Stable |
| `cuda11.8-py310` | 11.8 | 3.10 | Legacy PyTorch |
| `cpu-py311` | — | 3.11 | No GPU, CPU only |

## Build Cache

Zylora caches Docker layers between deployments. Layers above your `COPY` instructions are rebuilt only when they change.

<Callout type="info">
Pin your base image tag (e.g., `runner:cuda12.4-py311-2026.04`) for reproducible builds in production.
</Callout>

## Build Secrets

For private pip registries or model downloads requiring authentication:

```bash
zylora secret set HF_TOKEN=hf_...
```

```python
@zylora.function(gpu="A100-40GB", secrets=["HF_TOKEN"])
def run(prompt: str) -> str:
    import os
    os.environ["HF_TOKEN"]  # available at build + runtime
    ...
```
